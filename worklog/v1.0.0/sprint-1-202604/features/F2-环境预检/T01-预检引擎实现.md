# T01: 预检引擎实现

**优先级**: P0
**状态**: READY
**依赖**: F1/T01

## 目标
实现 bootstrap precheck 模块，检查目标环境是否满足 DTS 安装条件

## 技术设计

`internal/bootstrap/precheck/` 实现以下检查项：

```go
type CheckResult struct {
    Name    string      // 检查项名称
    Status  Status      // PASS / WARN / FAIL
    Message string      // 详细信息
    Detail  interface{} // 结构化数据
}

// 检查项列表:
// 1. K8s API 可达性（kubeconfig 有效，API server 响应）
// 2. K8s 版本兼容（>= 1.27）
// 3. 存储类可用（至少一个 default StorageClass）
// 4. 节点资源（总 CPU >= 4c, 总 Memory >= 8Gi, 磁盘 >= 50Gi）
// 5. DNS 解析（集群内 DNS 正常）
// 6. 命名空间可创建（dts-system, dts-packs）
// 7. RBAC 权限（当前 kubeconfig 有 cluster-admin 或等效权限）
```

`dts-bootstrap precheck` 子命令调用引擎，输出表格化报告，
任一 FAIL 则退出码非零，WARN 不阻塞。

## 影响范围
- `internal/bootstrap/precheck/checker.go` — 检查引擎
- `internal/bootstrap/precheck/checks.go` — 各检查项实现
- `cmd/bootstrap/main.go` — 注册 precheck 子命令

## 铁律合规

| Iron Law | Constraint | Detail |
|----------|-----------|--------|
| Law 3 (Data Security) | Precheck results must be classified as INTERNAL | CheckResult output may contain cluster topology, node IPs, resource capacities — classify as `INTERNAL` data level; do not expose in LLM context without sanitization |
| Law 2 (Security) | Precheck must verify Keycloak-compatible OIDC endpoint availability | Add check item #8: verify that a Keycloak-compatible OIDC endpoint is reachable (if configured) — status WARN if not found (prep for commander JWT validation); this is advisory in Sprint-1, not blocking |
| Ontology Prep | Precheck results map to infra/* ontology | CheckResult structures align with `infra/Cluster` and `infra/Node` ontology types; DataBinding to K8s API for real-time status |
